Power BI

Initially when I saw the task, I figured the best tool for the job was Power BI. Power BI allowed me to quickly import the data; and the interface made it very easy to create the required graphs. Unfortunately due to the nature of the application, it was impossible to export to a more web-friendly format due to the permissions Microsoft set on non-business users.

Angular, Firebase and Chart.js

After realising it may not be the best idea to soley submit a file with an unfamilar ''.pbix' format, I decided I'd attempt the problem using my preferred framework and cloud service: Angular & Firebase. I converted the CSV file to JSON and imported it into Firebase, my aim was to make the solution more fitting for a real-world problem. After connecting my Angular app to the database, I was able to manipulate the data to make it compitable with the Chart.js interface.

Dynamic Data Processing

I wanted to try and make the website interact seamlessly with the backend data as opposed to just having a static data source which can't be updated. I managed to make good progress and now, if any new records are added to the original job listings, the graphs will update to reflect the changes
